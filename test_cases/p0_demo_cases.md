# 智慧招采通 P0 演示测试示例

## 示例 1：评审模型与方案架构

用户在协作平台输入：

```text
@招采智囊 帮我分析这个示范项目采购文件和技术规范，构建评审模型，并输出应答方案结构。
```

附件：

- demo_tender_document.pdf
- demo_technical_spec.pdf

期望流程：

1. 解读采购文件。
2. 核验技术规范关键数据。
3. 提炼资格门槛和评审标准。
4. 构建评审计分模型。
5. 输出无偏离索引和佐证链目录。
6. 输出应答方案完整章节结构。

期望输出摘要：

```text
研判结论：需人工确认是否参与投标。
已输出：评审模型、客观分材料清单、无偏离索引、应答方案架构。
后续建议：由商务负责人确认资质、应标意向、报价和商务承诺。
```

## 示例 2：应标任务分解

用户在协作平台输入：

```text
@招采智囊 根据刚才的采购文件，帮我分解应标任务。
```

期望输出：

```text
任务清单：
1. 资料结构化处理和关键数据核验
2. 商务材料准备
3. 评审模型确认
4. 应答方案章节撰写
5. 图表和佐证材料补充
6. 报价确认
7. 合规审查
8. 签章和提交
```

## 示例 3：提交前合规审查

用户在协作平台输入：

```text
@招采智囊 帮我审查这个应答文件草稿有没有缺项、条款偏离、参考文件残留和模板化表达。
```

期望输出：

```text
研判结论：需补充材料。
发现问题：缺少项目负责人佐证材料，技术应答中有 2 处未逐条对应评审项，存在 1 处参考项目地点残留。
```

## 示例 4：双信封制项目

用户在协作平台输入：

```text
@招采智囊 这是双信封制招标文件，先帮我把技术标第一信封的内容结构规划出来。
```

期望输出：

```text
技术标第一信封内容结构：
1. 技术方案正文（不含报价信息）
2. 项目团队配置及佐证
3. 企业技术能力证明
4. 项目实施计划
5. 质量与安全保证措施
注意：第一信封不得出现任何价格、报价、折扣等商务信息。
```

## v2.0.0 新增测试用例

### 用例7: 评标方法深度分析
**输入**: 招标文件（综合评分法-固定分值法）
**预期输出**: 评审因素权重表 + 价格分策略 + 差异化报价建议

### 用例8: 废标风险扫描
**输入**: 投标文件初稿
**预期输出**: 废标风险清单(按严重度排序) + 修复建议

### 用例9: 电子招投标适配
**输入**: 招标公告（指定电子交易平台+远程异地评标）
**预期输出**: 平台适配检查清单 + CA互认确认 + 主场/副场机制说明

### 用例10: 中小企业优惠政策计算
**输入**: 企业信息（中小型企业）+ 招标文件（预留份额+价格扣除）
**预期输出**: 优惠政策适用分析 + 价格扣除计算 + 声明函模板

### 用例11: 救济机制时限计算
**输入**: 评标结果公示日期
**预期输出**: 质疑截止日 + 投诉截止日 + 诉讼截止日 时间轴
