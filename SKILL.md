# 智慧招采通 (Smart Procurement Navigator) v2.0.0

## 概述

智慧招采通是面向市场开拓、商务洽谈、方案策划、标书编制和经营决策层的智能招采协作系统。它贯穿从商机研判、评审建模、应答架构设计、章节写作到呈交前质量把关的完整业务链路，全面适配《中华人民共和国政府采购法》及《中华人民共和国招标投标法》框架下的政府采购、工程建设、服务类项目。

本技能与市场、商务、方案、标书和投标决策人协同，推动以下 12 大业务环节：
1. **商机甄别** — 接收采购公告或信息源，输出商机摘要和重要时间节点预警。
2. **投标研判** — 评估资质条件、过往业绩、预算匹配、交付能力和竞争格局。
3. **采购文件解读** — 提炼资格门槛、评审办法、呈交规范、否决投标风险点。
4. **资料结构化处理** — 将采购文件、技术规范、参考应答文件和知识库转为可检索文本。
5. **评审模型构建** — 区分否决性条款、主观/客观评分项、佐证材料和应答章节映射。
6. **应标任务分解** — 输出资料清单、责任人、截止时间和协作节点。
7. **应答方案架构设计** — 输出前置索引表、完整章节目录、章节定位和篇幅规划。
8. **应答文稿生成** — 根据评审要项、技术规范和知识库产出章节初稿。
9. **文稿润色与可视化增强** — 清除模板化表达、内部策略术语，并生成图表占位索引。
10. **合规性审查** — 检查缺项、格式规范、条款偏离、关键数据、参考文件残留和否决风险。
11. **呈交终审** — 提交前的最终一致性校验与签章材料核查。
12. **经验沉淀与知识复用** — 提炼中标/未中标根因，沉淀可复用的素材和策略。

## 招投标法律法规体系

### 核心法律框架

| 层级 | 法律文件 | 要点 |
|------|----------|------|
| 法律 | 《中华人民共和国招标投标法》（1999年通过，2017年修正） | 规范境内招标投标活动，必须招标范围：大型基础设施/公用事业项目、国有资金项目、国际组织贷款项目 |
| 法律 | 《中华人民共和国政府采购法》（2025年现行版） | 规范各级国家机关/事业单位/团体组织使用财政性资金采购。2025年要点：财政性资金范围扩大至政府投资基金、PPP项目 |
| 行政法规 | 《招标投标法实施条例》 | 配套实施细则 |
| 行政法规 | 《政府采购法实施条例》 | 第34条：评标方法分为最低评标价法和综合评分法 |
| 部门规章 | 《政府采购货物和服务招标投标管理办法》（财政部令第87号） | 第55条：评审因素应当细化和量化，与商务条件和采购需求对应 |
| 部门规章 | 《政府采购质疑和投诉办法》（财政部令第94号，2018年施行） | 质疑→投诉→复议/诉讼的完整程序 |
| 部门规章 | 《政府采购需求管理办法》 | 第21条：参与评分的指标应当是采购需求中的量化指标 |

### 2024-2025年重要政策文件

| 文件 | 核心内容 |
|------|----------|
| 国办发〔2024〕21号 | 创新完善体制机制推动招标投标市场规范健康发展 |
| 发改办法规〔2025〕807号 | 全国推广远程异地评标，13项具体措施，建立主场+副场协作机制 |
| 2025年专项整治 | 严厉打击采购人设置差别歧视条款、代理机构乱收费等行为 |

### 采购方式（政府采购法第26条）

| 方式 | 适用条件 |
|------|----------|
| **公开招标** | 主要采购方式，采购金额超法定限额 |
| **邀请招标** | 具有特殊性只能从有限供应商采购；公开招标费用占比过大 |
| **竞争性谈判** | 招标后无合格标；技术复杂无法确定详细规格；紧急需要；无法事先计算价格 |
| **单一来源采购** | 唯一供应商；不可预见紧急情况；添购金额≤原合同10% |
| **询价** | 货物规格标准统一、现货充足、价格变化幅度小 |

## 招投标全流程详解（7大阶段）

### 阶段1：招标前准备（基础保障阶段）

**核心目标**：明确采购需求、确定招标方式、完成内部审批

**关键步骤**：
1. **需求调研与确认**：联合使用部门明确规格型号、技术参数、数量、交付周期、质量标准、售后服务要求 → 输出《采购需求说明书》
2. **成本测算与预算审批**：财务部门根据市场行情测算合理预算，完成内部审批
3. **确定招标方式**：根据采购金额、标的特殊性、市场竞争程度选择合规方式
4. **组建招标团队**：采购（牵头）+ 技术（评审参数）+ 财务（评审价格）+ 法务（合规审核）+ 使用部门（确认需求）→ 输出《招标工作计划》

### 阶段2：招标文件编制与发布（信息公开阶段）

**核心目标**：编制合法、清晰的招标文件，避免歧义

**关键步骤**：
1. **编制招标文件**（核心文件）：
   - 招标公告：项目名称、采购内容、投标人资格要求、报名方式、文件获取时间/费用
   - 投标人须知：投标流程、截止时间、投标保证金要求（≤预算2%）、无效投标情形
   - 技术需求书：详细参数、质量标准、验收方式
   - 商务条款：合同主要条款、付款方式、交付地点、违约责任、报价要求
   - 评标办法：评分维度及权重（需客观可量化）
2. **内部评审与合规审核**：确认参数无倾向性、条款无法律风险
3. **发布公告**：公开招标公告期限≥5个工作日（国家指定平台+企业官网）
4. **答疑澄清**：截止前10天书面提问，截止前7天发布《答疑澄清公告》

**法定时限**：自招标文件发出之日起至投标截止日≥20日

### 阶段3：投标文件编制与提交（供应商响应阶段）

**核心目标**：供应商根据招标文件要求，提交完整、合规的投标文件

**投标文件三大组成部分**：

| 组成部分 | 权重 | 核心内容 |
|----------|------|----------|
| **商务标** | ~10% | 法定代表人身份证明、法人授权委托书（正本原件）、投标函、投标保证金凭证、营业执照、资质证书、安全生产许可证、信用查询报告、资信证明、三大体系认证、近三年财务状况表 |
| **技术标** | ~40% | 技术参数逐条应答、技术偏差表、技术资格证明材料、专项应答文件、过往业绩及证明材料、获奖情况、实施计划、服务保障方案 |
| **报价标** | ~50% | 报价表、分项报价表、报价说明、计价依据。不超过最高投标限价（红线），分项报价之和=总价 |

**投标文件提交**：密封完好，正副本各1份，投标截止时间前送达指定地点。逾期或未密封，招标方有权拒收。

### 阶段4：开标与评标（核心决策阶段）

**开标流程**：
1. 投标截止时间后立即开标
2. 检查投标文件密封情况
3. 公开宣读投标人名称、投标总价、保证金缴纳情况
4. 记录在《开标记录表》，投标人签字确认

**评标流程**：
1. **初步评审**（资格性+符合性审查）：审核资格证明文件 → 审核投标文件符合性 → 不符合的判定为"无效投标"
2. **详细评审**（打分排序）：按预设评标办法打分 → 汇总各评委分数 → 按总分从高到低排序 → 确定中标候选人（1-3名）
3. 出具《评标报告》，评标委员会签字确认

**评标委员会**：人数≥5人单数，专家≥2/3。独立打分，不得协商评分。与投标人有利害关系的需回避。

### 阶段5：定标与公示（结果确认阶段）

1. **定标**：通常选择排名第一的中标候选人（除非存在虚假投标、无履约能力等情形）
2. **中标公示**：≥3个工作日，公示内容包括中标候选人名单、得分、中标金额、异议渠道
3. **异议处理**：公示期内书面提出，招标方3日内答复
4. **发送中标通知书**：公示无异议后5个工作日内，具有法律效力

### 阶段6：合同签订与执行（落地阶段）

1. **签订合同**：中标通知书发出后30日内，合同条款与招标文件、投标文件实质性一致
2. **履约保证金**：≤合同金额10%，合同签订前缴纳，履约完毕后无息退还
3. **验收**：按招标文件约定的验收标准验收，签署《验收报告》

### 阶段7：归档复盘（闭环管理阶段）

1. **文件归档**：保存≥15年（政府采购法定要求），包括采购需求说明书、招标文件、投标文件、开标记录、评标报告、中标公示、合同、验收报告、付款凭证
2. **流程复盘**：输出《复盘报告》，优化后续规范

## 评标方法体系

### 综合评分法（最常用）

**核心法律要求**（《政府采购法实施条例》第34条）：
> 评审标准中的分值设置应当与评审因素的量化指标相对应。

**三种常见做法及合规性分析**：

| 做法 | 描述 | 合规性 |
|------|------|--------|
| **做法一** | 设定技术总分，根据技术参数条数及重要程度逐条对应赋分 | ✅ 合规（推荐） |
| **做法二** | 设定技术总分，每不满足一项扣除相应分值，扣完为止 | ❌ 违规（财政部多个案例认定：剩余量化指标未被赋予对应分值，无法区分10项以上负偏离产品的质量差异） |
| **做法三** | 不设扣分上限 | ❌ 违规（可能导致负分或超总分，违反一一对应原则） |

**评审因素设置最佳实践**：
1. **底线参数**：设为符合性条件，一票否决（减少需赋分参数数量）
2. **重要参数**：赋予高于一般参数的分值（体现优质产品优势）
3. **一般条件**：归并处理，避免逐条拆分为数百个参数导致单条分值过低
4. **验证**：确认量化指标与量化分值一一对应

### 最低评标价法

- 适用于技术标准统一、通用性强的货物或服务
- 以经评审的最低投标价为优先
- 技术、服务等标准统一的货物和服务项目，应当采用最低评标价法

### 其他评标方法

| 方法 | 适用场景 | 核心逻辑 |
|------|----------|----------|
| **经评审的最低投标价法** | 考虑价格调整因素 | 经评审后的最低价中标 |
| **性价比法** | 政府采购 | 总分÷投标报价，比值最高者中标 |
| **合理低价法** | 工程类 | 剔除异常报价后的最低价 |
| **两阶段评标法** | 技术复杂项目 | 先技术后商务，分阶段评审 |

## 电子招投标平台适配

### 全流程电子化

电子招投标平台覆盖：招标公告发布 → 招标文件编制 → 投标 → 开标 → 评标专家抽取 → 异议澄清 → 定标 → 合同签订 → 文件归档

**平台核心功能**：
- 招标文件结构化编制（减少人为错误）
- 投标文件制作工具
- CA数字证书（加密、解密、电子签章）
- 远程异地评标
- 暗标评审（隐藏投标人信息）
- 智能辅助评标（客观分自动计算、投标文件自动定位、信息智能验真）
- 评定分离
- 多种评标方法支持

### 远程异地评标（发改办法规〔2025〕807号）

**核心机制**：主场+副场协作

| 角色 | 职责 |
|------|------|
| **主场** | 提供电子交易系统与专业交易工具，负责行政监督主责 |
| **副场** | 配合专家抽取、身份核验、评标、报告签署、资料保存 |
| **资源共享** | 评标专家库共享目录 + 评标场所工位共享目录 |
| **成本分担** | 主副场协商确定补偿机制，收费公开透明 |
| **专家报酬** | 鼓励"就高不就低"原则 |

### 评定分离

- **评标阶段**：专家按规则评标，推荐不排序的中标候选人
- **定标阶段**：招标人组建定标委员会，择优确定中标人
- 落实招标人主体责任，实现权责对等

## 投标文件编制深度指南

### 商务标编制要点

**核心使命**：证明"公司有资格、有能力承接项目"（门槛作用）

**必备清单**：
1. 法定代表人身份证明
2. 法人授权委托书（正本必须是原件）
3. 投标函及投标函附录
4. 投标保证金交存凭证复印件
5. 对招标文件及合同条款的承诺及补充意见
6. 企业营业执照、资质证书、安全生产许可证（注意有效期）
7. 信用查询报告（"信用中国"等官方平台）
8. 资信证明（银行出具）
9. 三大体系认证（质量管理、环境管理、职业健康安全）
10. 近三年财务状况表（附审计报告）

**常见错误**：资质过期、业绩合同缺少金额页、授权委托书为复印件而非原件

### 技术标编制要点

**核心使命**：证明"公司有足够技术能力，能拿出完全符合采购要求的产品或服务"（核心竞争力，约40%权重）

**必备内容**：
1. 项目货物技术特性的逐条应答文件（不要只写"符合"，要结合具体参数详细说明）
2. 技术偏差表（有偏差如实说明，不要模糊表述）
3. 支撑技术资格审查的证明材料（专利、技术人员资质证书等）
4. 专项应答文件（针对特殊技术要求）
5. 过往业绩及证明材料（优先选类似项目案例，附合同关键页+验收报告）
6. 获奖情况（行业内权威奖项）
7. 实施计划（甘特图/里程碑计划）
8. 服务保障体系（质保期、响应时间、运维团队配置）

**常见错误**：技术方案与招标文件核心需求脱节、参数应答模糊、业绩案例不匹配

### 报价标编制要点

**核心使命**：报价的合理性与竞争力（定胜负手，约50%权重）

**必备内容**：
1. 报价表（分项报价+总价）
2. 报价说明（计价依据、是否含税、税率）
3. 分项报价表（设备采购/安装调试/售后培训等分项）

**关键原则**：
- 不超过最高投标限价（超了直接废标，红线）
- 分项报价之和=总价（避免计算错误）
- 考虑成本+合理利润+行业市场价
- 报价需有竞争力但不低于成本（低于成本需提供说明）

## 常见废标原因及规避策略

### 资格性废标

| 原因 | 规避策略 |
|------|----------|
| 资质证书过期 | 建立资质有效期台账，提前3个月预警续期 |
| 业绩不满足要求 | 仔细核对招标文件业绩要求（金额、类型、时间），选择最匹配案例 |
| 财务状况不符合 | 提前准备审计报告，确保财务指标达标 |
| 经营范围不匹配 | 检查营业执照经营范围是否覆盖项目内容 |

### 符合性废标

| 原因 | 规避策略 |
|------|----------|
| 未响应实质性条款（★条款） | 逐条核对招标文件中标注★的条款，确保完全响应 |
| 技术参数重大偏离 | 技术偏差表如实填写，核心参数不可偏离 |
| 报价超过最高限价 | 报价前确认拦标价，设置内部审核机制 |
| 投标有效期不足 | 确保投标有效期≥招标文件要求 |

### 形式性废标

| 原因 | 规避策略 |
|------|----------|
| 逾期提交 | 至少提前1天送达，预留缓冲时间 |
| 未密封或密封不规范 | 按招标文件要求封装，加盖骑缝章 |
| 签字盖章不全 | 制作签字盖章检查清单，逐项核对 |
| 投标保证金不足或未到账 | 提前3个工作日办理，保留凭证 |
| 格式不符合要求 | 严格按招标文件提供的格式模板填写 |

## 质疑/异议/投诉三级救济机制

### 政府采购路径（质疑→投诉→复议/诉讼）

| 环节 | 时限 | 受理方 | 法律依据 |
|------|------|--------|----------|
| **质疑** | 知道权益受损之日起**7个工作日内** | 采购人/代理机构 | 《政府采购法》第52条 |
| 质疑答复 | 收到后**7个工作日内** | 采购人/代理机构 | 《政府采购法》第53条 |
| **投诉** | 答复期满后**15个工作日内** | 同级财政部门 | 《政府采购法》第55条 |
| 投诉受理决定 | 收到后**3个工作日内** | 财政部门 | 《政府采购供应商投诉处理办法》 |
| 投诉处理 | 受理后**30个工作日内** | 财政部门 | 《政府采购供应商投诉处理办法》 |
| **行政复议** | 收到决定后**60日内** | 上级机关 | 《行政复议法》 |
| **行政诉讼** | 收到决定后**6个月内** | 人民法院 | 《行政诉讼法》 |

**注意**：质疑是投诉的前置程序——必须先向采购人/代理机构提出书面质疑，对答复不满意或未答复才能投诉。

### 招标投标路径（异议→投诉→复议/诉讼）

| 环节 | 时限 | 受理方 | 法律依据 |
|------|------|--------|----------|
| 异议（招标文件） | 投标截止**10日前** | 招标人 | 《招标投标法实施条例》第54条 |
| 异议（开标） | **当场**提出 | 招标人 | 《招标投标法实施条例》 |
| 异议（评标结果） | **公示期间** | 招标人 | 《招标投标法实施条例》 |
| 异议答复 | 收到后**3日内** | 招标人 | 《招标投标法实施条例》 |
| **投诉** | 知道权益受损之日起**10日内** | 行政监督部门 | 《工程建设项目招标投标活动投诉处理办法》 |
| 投诉受理决定 | 收到后**3个工作日内** | 行政监督部门 | 同上 |
| 投诉处理 | 受理后**30个工作日内** | 行政监督部门 | 同上 |

**注意**：对资格预审文件、招标文件、开标、评标结果投诉前，必须先向招标人提出异议——异议是投诉的前置程序。

### 投诉处理结果类型

| 情形 | 处理结果 |
|------|----------|
| 采购文件具有歧视性/倾向性 | 责令修改采购文件，重新开展采购活动 |
| 采购过程影响中标结果，合同未签 | 决定采购行为违法，责令重新采购 |
| 合同已签但未履行 | 撤销合同，重新采购 |
| 合同已履行 | 决定采购活动违法，相关责任人赔偿 |
| 虚假/恶意投诉 | 列入不良记录，禁止1-3年参加政府采购 |

### 证据准备要点

- 必备证据：招标文件及补充文件、投标文件副本、开标记录、中标公示截图
- 针对性证据：评委评分表（质疑评标不公）、投标文件异常相似点（投诉串标围标）、IP地址相同证据
- 程序证据：质疑函、快递签收记录、邮件回执
- **严禁**：伪造检测报告、业主证明等（一经查实列入黑名单）

## 中国招投标特色场景支持

### 双信封制（技术标/商务标分离评审）

- 第一信封（技术标）：先评审技术方案，技术标不合格的投标文件不再开启第二信封
- 第二信封（商务标/报价）：技术标合格的投标人进入商务评审
- 适用场景：技术复杂、方案差异性大的项目
- 支持技术标第一信封和商务标第二信封的独立编制、评审模型和封装检查

### 资格预审 vs 资格后审

| 维度 | 资格预审 | 资格后审 |
|------|----------|----------|
| 时机 | 招标前 | 开标后 |
| 方式 | 合格制/有限数量制 | 通过/不通过 |
| 优点 | 减少评标工作量，提前筛选 | 缩短招标周期，增加竞争 |
| 缺点 | 周期长，可能泄露潜在投标人信息 | 评标工作量大，不合格者浪费投标成本 |
| 适用 | 大型复杂项目、潜在投标人较多 | 中小型项目、投标人较少 |

### 中小企业优惠政策

- **价格扣除**：小型/微型企业投标报价给予6%-10%的价格扣除（用扣除后的价格参与评审）
- **预留份额**：≥30%的预算面向中小企业采购，其中≥60%预留给小微企业
- **依据**：《政府采购促进中小企业发展管理办法》
- **操作**：识别采购文件中面向中小微企业的价格扣除、预留份额和优先采购条款

### 联合体投标

- **资质认定**：按资质等级较低的单位确定联合体资质等级
- **连带责任**：联合体各方对采购人承担连带责任
- **协议要求**：需提交联合体协议，载明各方承担的工作和义务
- **基本条件**：联合体各方均需满足《政府采购法》第22条规定的基本条件

### 两阶段招标

- **第一阶段**：技术方案征询——投标人提交不带报价的技术建议，招标人明确技术方案和标准
- **第二阶段**：报价投标——招标人向第一阶段提交技术建议的投标人提供招标文件，投标人提交最终技术方案和报价
- 适用场景：技术复杂、无法精确拟定技术规格的项目

## 工作原则与边界

### 工作原则
1. **合规底线**：不可遗漏资格条件、评审指标、呈交格式、否决条款和时间节点。
2. **溯源明确**：全部风险提示须标注采购文件原文位置、章节或条款编号。
3. **评审导向**：先构建评审计分模型，再规划章节架构和写作深度。
4. **项目化撰写**：技术方案须紧密围绕采购需求、技术标准、现场条件和项目实际，禁止直接搬用参考项目文件。
5. **严禁虚构**：不可编造企业资质、人员证书、项目业绩、报价或交付保证。
6. **表达克制**：正式应答文件中排除机械模板化表达、夸张修饰词和内部评审策略用语。
7. **结构化输出**：优先采用清单、对比表、目录树和风险等级矩阵呈现。
8. **保密与时效**：涉及客户信息、报价、资质和历史应答文件的，按授权范围展示；对报名、答疑、保证金和递交截止等关键时间点须重点标示。

### 行为边界（不可为）
- **决策留给人**：可提供推进分析建议，但不得替代责任主体做出最终投标决策（不可自动报价、不可代确认商务承诺、不可代盖章和代提交）。
- **内容净化**：不可在正式应答文件中保留"得高分""绝对优势""远超同行"等内部判断用语；不可将参考项目中的其他项目称谓、地点、特定型号、工期或金额带入目标项目。

## 核心业务流程

### 1. 商机甄别
- **流程**：接收采购公告/信息来源/截图 → 抽取项目名称、采购主体、预算额度、报名与递交截止日期 → 判定行业归属、地域范围、预算适宜度和业务契合度 → 输出商机摘要 → 设定报名、答疑、保证金和递交截止提醒。
- **输出**：商机摘要、时间节点表、初步推进意见、待补充资料。

### 2. 投标研判
- **流程**：读取商机摘要、采购文件和企业能力素材 → 提取资格准入条件 → 匹配企业资质、历史业绩、人员配置和交付实力 → 评估预算合理性、履约周期、地理范围、竞争态势和履约风险 → 标定报价、商务承诺和最终投标决策的人工确认点。
- **输出**：研判报告（积极跟进 / 谨慎评估 / 建议放弃）、资质能力缺口清单、风险评级、决策确认项。

### 3. 采购文件解读
- **流程**：上传/读取采购文件 → 解析文档结构体系 → 萃取章节与核心要求 → 提炼资格门槛 → 提炼评审标准与分值分布 → 提炼呈交格式、签章规范、份数要求、电子平台和保证金条款 → 生成待澄清疑问 → 输出采购文件解读报告。
- **输出**：项目概要、资格条件矩阵、评审标准明细、呈交要求清单、否决条款、澄清问题列表。

### 4. 资料结构化处理
- **流程**：接收采购文件、技术规范、参考应答文件和知识库 → 核验文件是否已完成可检索文本转换 → 抽取评审表、技术参数表、作业指标、履约周期和关键数据 → 标定扫描件、断列表格、缺页、图片文字和不可检索段落 → 输出资料索引和补转换清单。
- **输出**：资料结构索引、关键数据清单、残缺/断损段落标记、需人工补录内容。

### 5. 评审模型构建
- **流程**：读取评审标准、资格门槛和呈交要求 → 建立评审模块、分值权重、评分性质和应答策略 → 区分否决性门槛和评审打分项 → 识别客观分佐证材料 → 输出否决性条款无偏离索引 → 输出评审得分索引总表和履约承诺佐证链目录。
- **输出**：评审计分模型表、客观分材料清单、无偏离索引、佐证链目录。

### 6. 应标任务分解
- **流程**：读取解读报告、评审模型和无偏离索引 → 分解商务、技术、报价、资质、签章和上传任务 → 按评审项权重和否决风险标注优先级 → 分配责任主体和截止时间 → 输出待办清单和节点提醒。
- **输出**：应标任务清单、里程碑、责任人、截止时间、风险优先级矩阵。

### 7. 应答方案架构设计
- **流程**：读取评审模型、技术规范和过往应答文件 → 对比参考文件，识别可借鉴要素和须排除内容 → 输出完整章节结构 → 在正文前部设置前置索引表 → 为每章产出"本章对应评审项""本章写作锚点""本章成果佐证" → 标注重点章节篇幅和资料需求。
- **输出**：完整章节目录、前置索引表、章节写作锚点、篇幅规划、素材需求。

### 8. 应答文稿生成
- **流程**：读取应答方案架构和技术规范 → 按章节展开正文内容 → 将技术参数、范围、指标、规格和工期嵌入对应章节 → 展开专项执行细则、标准作业流程、月度分解计划、风险预案和验收资料包 → 标定需人工补充的案例、证书、报价和承诺。
- **输出**：章节初稿、待补充材料、佐证链缺口、人工确认项。

### 9. 文稿润色与可视化增强
- **流程**：读取章节初稿 → 检测并替换模板化机械用语、内部评审策略词、夸张修辞和不符合政府采招文件规范的表达 → 输出替换对照建议 → 生成图表占位索引 → 为技术路线图、工艺流程示意图、进度甘特图、组织架构图和响应流程图输出占位说明。
- **输出**：润色报告、替换建议、图表索引、图表占位清单。

### 10. 合规性审查
- **流程**：上传应答文件草稿或终稿 → 核查商务材料缺漏 → 核查格式、签章、页序、编码和呈交规范 → 核查技术/商务应答偏离和前后矛盾 → 核查项目称谓、编号、采购主体、地点、履约期限和关键数据一致性 → 检索参考文件残留和模板化表达禁用词 → 输出提交前合规审查报告。
- **输出**：审查结论、否决级风险、一般性问题、整改方案、终审确认清单。

### 11. 呈交终审
- **流程**：读取终稿与合规审查报告 → 逐项核验签章、证书原件、电子文档格式 → 核对电子招投标平台上传要求 → 确认保证金凭证和报价密封规范 → 输出终审放行单。
- **输出**：终审放行单、签章核对表、上传确认清单。

### 12. 经验沉淀与知识复用
- **流程**：输入应标结果、过程记录和审查报告 → 汇总关键决策节点和问题 → 分析中标或未中标深层原因 → 沉淀评审模型、高分段、风险清单和可复用素材 → 更新知识库和模板库。
- **输出**：复盘报告、可复用素材、模板优化建议、下次应标改进清单。

## 典型项目案例库

### 案例1：政府信息化采购（综合评分法）

| 维度 | 内容 |
|------|------|
| **项目类型** | 政务云平台建设项目 |
| **采购方式** | 公开招标 |
| **预算金额** | 800万元 |
| **评标方法** | 综合评分法（价格30%+技术50%+商务20%） |
| **关键资格** | CMMI 3级、ISO 27001、等保三级测评资质 |
| **核心评审点** | 云平台架构设计（15分）、数据安全方案（10分）、国产化适配（10分）、运维SLA（8分）、同类项目业绩（7分） |
| **典型否决项** | 未提供等保测评资质、技术方案缺少灾备设计 |
| **中标关键** | 技术方案深度（详细到组件级设计）+ 3个同类省级政务云案例 |
| **经验教训** | 报价权重虽仅30%，但价格分差可拉开10分以上，需精准测算 |

### 案例2：工程建设项目（经评审的最低投标价法）

| 维度 | 内容 |
|------|------|
| **项目类型** | 产业园区道路及管网工程 |
| **采购方式** | 公开招标 |
| **预算金额** | 3500万元 |
| **评标方法** | 经评审的最低投标价法 |
| **关键资格** | 市政公用工程施工总承包二级及以上、安全生产许可证 |
| **核心评审点** | 技术方案通过性审查（合格制），通过后以经评审的最低投标价中标 |
| **典型否决项** | 资质等级不满足、安全生产许可证过期、投标报价超过拦标价 |
| **中标关键** | 精准成本测算（不低于成本）+ 技术方案通过审查 |
| **经验教训** | 最低价法下价格是唯一竞争维度，成本控制能力决定竞争力 |

### 案例3：医疗设备采购（综合评分法+双信封制）

| 维度 | 内容 |
|------|------|
| **项目类型** | 三甲医院影像设备采购 |
| **采购方式** | 公开招标（双信封制） |
| **预算金额** | 1200万元 |
| **评标方法** | 综合评分法（技术60%+价格40%），双信封 |
| **关键资格** | 医疗器械经营许可证、设备厂家授权书、FDA/CE认证 |
| **核心评审点** | 设备技术参数（25分）、临床应用效果（15分）、售后服务（10分）、培训方案（5分）、耗材成本（5分） |
| **典型否决项** | 第一信封技术标不合格直接淘汰，不开第二信封 |
| **中标关键** | 技术参数逐条响应+临床应用案例+本地化售后服务团队 |
| **经验教训** | 双信封制下技术标是第一关，技术方案必须过硬才能进入价格竞争 |

### 案例4：IT服务采购（竞争性磋商）

| 维度 | 内容 |
|------|------|
| **项目类型** | 企业ERP系统实施服务 |
| **采购方式** | 竞争性磋商 |
| **预算金额** | 500万元 |
| **评标方法** | 综合评分法（技术方案40%+演示30%+价格30%） |
| **关键资格** | 同类ERP实施案例≥3个、项目经理PMP认证 |
| **核心评审点** | 方案演示（30分）、实施方法论（15分）、行业经验（10分）、团队配置（10分）、风险应对（5分） |
| **典型否决项** | 演示环节无法展示核心功能、实施计划严重脱离实际 |
| **中标关键** | 现场演示效果+行业标杆案例+实施团队资质 |
| **经验教训** | 竞争性磋商可以多轮报价，首轮报价需留谈判空间 |

## 行业特定注意事项

### 政府信息化项目
- 优先支持国产化（信创目录内的CPU/OS/数据库/中间件）
- 数据安全：需满足等保+密码应用安全性评估
- 接口开放：需承诺开放API/数据接口
- 本地化服务：需证明本地服务团队能力

### 工程建设类项目
- 项目经理不得有在建项目（需提供无在建承诺书）
- 农民工工资保证金制度
- 安全文明施工费不可竞争
- 暂列金额/暂估价按规定计取

### 医疗设备采购
- 进口设备需提供进口论证审批文件
- 耗材报价需单独列示并计入评审
- 临床应用培训需提供详细方案
- 设备兼容性需提供技术证明

### IT服务类项目
- 人员稳定性承诺（核心人员不得更换）
- 知识转移计划（项目结束后移交）
- 源代码交付/托管条款
- SLA服务水平承诺（响应/解决时效）
