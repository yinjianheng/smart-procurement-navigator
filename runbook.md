# 智慧招采通 Runbook

## 一、部署前准备

1. 确认部署环境可运行智能代理平台。
2. 确认协作平台已开通并完成配置。
3. 准备示范采购文件、技术规范文本、过往应答文件模板、企业资质和项目案例。
4. 确认 `agent.yaml` 中的配置与实际环境一致。
5. 准备评审模型模板、应答方案结构模板和合规审查清单。

## 二、启动检查

```text
1. 启动智能代理
2. 加载 agent.yaml
3. 加载 procurement_skills.yaml
4. 连接协作平台
5. 执行 test_cases/p0_demo_cases.md 中的样例 1
6. 检查评审模型、应答方案结构和审查报告是否可正常产出
7. 检查日志是否正常写入
```

## 三、演示前检查

- 协作平台账号已登录。
- 智慧招采通会话窗口已固定。
- 演示采购文件已就绪。
- 演示技术规范或需求文件已就绪。
- 评审模型和应答方案完整章节结构模板已加载。
- 演示录屏文件可正常播放。
- 输出模板可正常渲染。

## 四、故障处理

| 问题 | 处理方式 |
| --- | --- |
| 文件解析异常 | 切换为预生成解析结果 |
| 文件体积过大 | 使用节选版演示文件 |
| 输出不稳定 | 使用固定提示词和录屏方案 |
| 风险项缺少溯源引用 | 切换为人工标注版演示结果 |
| 参考文件残留误报 | 在知识库中补充参考项目名称、地点、型号和工期排除清单 |
| 文稿润色误伤正式术语 | 将正式术语加入白名单并保留人工复核环节 |

## 五、生产化待办

- 对接采购公告来源或人工转发入口。
- 整理企业资质库、案例库和历史应答文件库。
- 建立评审项和否决风险检查规则集。
- 建立评审模型、无偏离索引和佐证链目录的标准模板。
- 建立模板化表达禁用词、参考文件残留和关键数据一致性检查规则。
- 增强应答方案完整章节结构、图表占位和章节展开能力。
- 对接协作平台待办事项，生成应标任务协同看板。
- 增强应标经验沉淀和素材归档流程。
- 适配各地公共资源交易中心电子招投标系统的报送规范。
- 建立政府采购法规更新追踪机制。


## v2.0.0 新增操作指南

### 评标方法深度分析操作
1. 识别招标文件中采用的评标方法
2. 判断综合评分法具体做法（固定分值/区间赋分/排序赋分）
3. 分析价格分计算公式（均价基准/最低价基准/最高限价基准）
4. 识别价格分异常场景（恶意低价/围标串标嫌疑）
5. 制定差异化报价策略

### 电子招投标操作清单
- [ ] 确认CA证书有效期
- [ ] 验证电子交易平台兼容性
- [ ] 远程异地评标：确认主场/副场安排
- [ ] 电子签章准备
- [ ] 加密投标文件制作
- [ ] 电子保函/保证金缴纳

### 废标预防检查清单
- [ ] 投标函签署盖章是否完整
- [ ] 投标保证金是否按时足额缴纳
- [ ] 投标有效期是否响应
- [ ] 技术规格是否实质性响应（无重大偏离）
- [ ] 报价是否在最高限价内
- [ ] 联合体协议是否完整（如适用）
- [ ] 中小企业声明函是否合规（如适用）
