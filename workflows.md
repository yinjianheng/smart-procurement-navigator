# 智慧招采通 Workflows

## 一、商机甄别

目标：将采购公告、链接或截图转化为结构化商机摘要，并标记关键时间节点。

```text
用户发送公告/信息源/截图
  -> ZC01 接收商机信息
  -> ZC02 抽取项目名称、采购主体、预算、报名和递交截止时间
  -> 判定行业归属、地域范围、预算适宜度和业务契合度
  -> 输出商机摘要
  -> ZC14 设定报名、答疑、保证金和递交截止提醒
```

输出：商机摘要、时间节点表、初步推进意见、待补充资料。

## 二、投标研判

目标：判断是否建议进入应标准备，并列出决策人必须确认的事项。

```text
读取商机摘要、采购文件和企业能力素材
  -> ZC04 提炼资格准入条件
  -> ZC08 匹配企业资质、历史业绩、人员配置和交付实力
  -> ZC07 评估预算合理性、履约周期、地域、竞争和履约风险
  -> 标定报价、商务承诺和最终应标决策的人工确认点
```

输出：积极跟进 / 谨慎评估 / 建议放弃、能力缺口清单、风险评级、决策确认项。

## 三、采购文件解读

目标：提炼采购文件中的资格门槛、评审标准、呈交规范和否决风险。

```text
上传采购文件
  -> ZC05 解析文档结构体系
  -> ZC03 萃取章节和核心要求
  -> ZC04 提炼资格门槛
  -> ZC05 提炼评审标准
  -> ZC06 提炼呈交格式、签章、份数、电子平台和保证金条款
  -> ZC15 生成待澄清疑问
  -> 输出采购文件解读报告
```

输出：项目概要、资格门槛矩阵、评审标准明细、呈交要求清单、否决条款、澄清问题列表。

## 四、资料结构化处理

目标：将采购文件、技术规范、参考应答文件和知识库转为可全文检索的结构化资料。

```text
接收采购文件、技术规范、参考应答文件和知识库
  -> ZC17 核验文件是否已完成可检索文本转换
  -> 抽取评审表、技术参数表、作业指标、履约期限和关键数据
  -> 标定扫描件、断列表格、缺页、图片文字和不可检索段落
  -> 输出资料结构索引和补转换清单
```

输出：资料结构索引、关键数据清单、残缺/断损段落标记、需人工补录内容。

## 五、评审模型构建

目标：建立"此项目凭什么得分"的准确模型。

```text
读取评审标准、资格门槛和呈交要求
  -> ZC18 建立评审模块、分值权重、评分性质和应答策略
  -> 区分否决性门槛和评审打分项
  -> 识别客观分佐证材料
  -> ZC19 输出否决性条款无偏离索引
  -> 输出评审得分索引总表和履约承诺佐证链目录
```

输出：评审模型表、客观分材料清单、无偏离索引、佐证链目录。

## 六、应标任务分解

目标：将采购要求拆解为可执行的应标协作任务。

```text
读取解读报告、评审模型和无偏离索引
  -> ZC09 分解商务、技术、报价、资质、签章和上传任务
  -> 按评审项权重和否决风险标注优先级
  -> 分配责任主体和截止时间
  -> ZC14 生成协作待办和节点提醒
```

输出：应标任务清单、里程碑、责任人、截止时间、风险优先级矩阵。

## 七、应答方案架构设计

目标：在动笔前完成应答方案骨架、章节锚点和篇幅规划。

```text
读取评审模型、技术规范和过往应答文件
  -> ZC20 对比参考文件，识别可借鉴要素和须排除内容
  -> ZC21 输出完整章节结构
  -> 在正文前部设置前置索引表
  -> 为每章产出"本章对应评审项""本章写作锚点""本章成果佐证"
  -> 标注重点章节篇幅和资料需求
```

输出：完整章节目录、前置索引表、章节写作锚点、篇幅规划、素材需求。

## 八、应答文稿生成

目标：基于评审模型、技术规范和知识库产出项目化章节初稿。

```text
读取应答方案架构和技术规范
  -> ZC22 按章节展开正文内容
  -> 将技术参数、范围、指标、规格和工期嵌入对应章节
  -> 展开专项执行细则、标准作业流程、月度分解计划、风险预案和验收资料包
  -> 标定需人工补充的案例、证书、报价和承诺
```

输出：章节初稿、待补充材料、佐证链缺口、人工确认项。

## 九、文稿润色与可视化增强

目标：提升正式应答文件质感，并增加可核验的图表表达。

```text
读取章节初稿
  -> ZC23 检测模板化机械表达、内部策略用语、夸张修饰和不适用于政府采购文件的行文
  -> 输出替换对照建议和仍需人工审核的句子
  -> ZC24 输出图表占位索引
  -> 为技术路线图、工艺流程示意图、进度甘特图、组织架构图和响应流程图输出占位说明
```

输出：润色报告、替换建议、图表索引、图表占位清单。

## 十、合规性审查

目标：呈交前检查缺项、条款偏离、格式规范、一致性、参考残留和否决风险。

```text
上传应答文件草稿或终稿
  -> ZC12 核查商务材料缺漏
  -> ZC13 核查格式、签章、页序、编码和呈交规范
  -> ZC14 核查技术/商务应答偏离和前后矛盾
  -> ZC25 核查项目称谓、编号、采购主体、地点、履约期限和关键数据一致性
  -> 检索参考文件残留和模板化表达禁用词
  -> 输出提交前合规审查报告
```

输出：审查结论、否决级风险、一般性问题、整改方案、终审确认清单。

## 十一、呈交终审

目标：最终呈交前逐项核验签章、证书、电子文档和保证金。

```text
读取终稿和合规审查报告
  -> 逐项核验签章、证书原件、电子文档格式
  -> 核对电子招投标平台上传规范
  -> 确认保证金凭证和报价密封规范
  -> 输出终审放行单和签章核对表
```

输出：终审放行单、签章核对表、上传确认清单。

## 十二、经验沉淀与知识复用

目标：应标结束后沉淀经验、素材和可复用的方法论。

```text
输入应标结果、过程记录和审查报告
  -> ZC16 汇总关键决策节点和问题
  -> 分析中标或未中标深层原因
  -> 沉淀评审模型、高分段、风险清单和可复用素材
  -> 更新知识库和模板库
```

输出：复盘报告、可复用素材、模板优化建议、下次应标改进清单。

## v2.0.0 新增工作流

### 电子招投标平台适配流程
1. 识别招标公告中指定的电子交易平台
2. 确认CA数字证书互认范围
3. 远程异地评标：识别主场/副场，确认专家抽取机制
4. 电子签章与加密投标文件制作

### 救济机制操作流程
```
收到不利结果
  → 7日内: 提出质疑（向采购人/代理机构）
  → 质疑答复不满意
    → 15日内: 向监管部门投诉
    → 投诉处理不满意
      → 60日内: 申请行政复议
      → 或6个月内: 提起行政诉讼
```

### 中小企业优惠政策应用流程
1. 确认企业类型（工信部联企业〔2011〕300号）
2. 声明函准备（《中小企业声明函》）
3. 价格扣除计算（6-10%）
4. 份额预留项目识别
5. 联合体投标方案设计
